# Visual regression checklist для leontev.pro

Этот чек-лист нужно проходить после каждой заметной правки главной, форматов, кейсов или метода.

## Mobile QA

Проверять ширины: 375, 390, 414, 430, 768 px.

- Нет horizontal scroll: `document.documentElement.scrollWidth <= window.innerWidth + 2`.
- Header не ломается, логотип не показывает broken image.
- На главной сначала читается оффер, потом портрет/доказательства, затем детали.
- Быстрые чипы/навигация не создают общий горизонтальный скролл страницы.
- Карточки не выходят за экран.
- Цены не распадаются на случайные строки.
- Термины `консьерж-сервис`, `дорожная карта`, `карта 360°` не выглядят разорванными без необходимости.
- Кнопки видимы, не белые без текста, не перекрывают контент.
- Дорожная карта на mobile — вертикальная линия, не 8 узких карточек.
- Таблицы и длинные сравнения должны скроллиться внутри своего контейнера, а не расширять страницу.
- Декоративные элементы, которые мешают чтению, скрыты или приглушены.

## Desktop QA

Проверять ширины: 1280, 1440, 1600, 1920 px.

- Главная выглядит как единая воронка, а не набор отдельных презентаций.
- Roadmap на главной: 4 + 4 карточки, не 8 в одну строку.
- Форматы на главной: 3 карточки в первом ряду, 2 широкие во втором.
- Статы “Цена хаоса”: 2 × 2, без вылезания текста.
- Блок “Цена хаоса”: `2–4 просмотра`, не `2–4 лишних просмотра`.
- Кнопки в карточках форматов стоят по нижнему краю.
- Нет пустых кнопок и низкоконтрастных кнопок.
- Нет broken logo / broken image в header.
- Нет бледного основного текста на бежевом фоне.
- Footer содержит основные маршруты и контакты.

## Автоматические проверки в консоли

Пустые кнопки:

```js
[...document.querySelectorAll('a, button')]
  .filter(el => el.className.toString().includes('btn') && !el.textContent.trim())
```

Должен вернуться пустой массив.

```js
[...document.querySelectorAll('a.button, button.button, a.btn, button.btn, a.channel, button.channel')]
  .filter(el => !el.textContent.trim())
```

Должен вернуться пустой массив.

Ссылки-кнопки без `href`:

```js
[...document.querySelectorAll('a.button, a.btn, a.channel')]
  .filter(el => !el.getAttribute('href'))
```

Должен вернуться пустой массив.

Horizontal scroll:

```js
document.documentElement.scrollWidth > window.innerWidth + 2
```

Должно быть `false`.

Проблемные элементы overflow:

```js
[...document.querySelectorAll('*')]
  .filter(el => el.scrollWidth > document.documentElement.clientWidth + 2)
  .map(el => ({ tag: el.tagName, class: el.className, width: el.scrollWidth }))
```

Должен быть пустой массив или только заведомо внутренний горизонтальный скролл навигации.

Broken images:

```js
[...document.images]
  .filter(img => img.complete && img.naturalWidth === 0)
  .map(img => img.src)
```

Должен вернуться пустой массив.

## Терминология

Использовать:

- консьерж-сервис при покупке квартиры;
- консьерж-сопровождение покупки;
- один контакт;
- финальные варианты;
- рабочие сценарии;
- управляемый маршрут;
- полная стоимость покупки.

Не использовать как основной термин:

- buyer concierge;
- property concierge;
- shortlist;
- подбор ссылок;
- агент со ссылками.
