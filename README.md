# Лендинг Павла Леонтьева

Статический одностраничный сайт для GitHub Pages. Сборка и зависимости не нужны: откройте `index.html` в браузере или опубликуйте репозиторий через Pages.

## Как заменить фотографию

1. Подготовьте квадратную фотографию в формате JPG.
2. Положите её в корень репозитория с точным именем `pavel-photo.jpg`.
3. Обновите страницу в браузере с очисткой кэша.

Если файла нет или он не загрузился, в круглом блоке автоматически появятся инициалы «ПЛ», поэтому вёрстка не ломается.

## Как включить GitHub Pages

1. Откройте репозиторий на GitHub: **Settings → Pages**.
2. В разделе **Build and deployment** выберите **Deploy from a branch**.
3. Укажите ветку **main** и папку **/(root)**, затем нажмите **Save**.
4. После публикации сайт будет доступен по адресу: <https://abkx12-hub.github.io/leontev-site/>.

## Подключение домена leontev.pro

Файл `CNAME` в корне уже содержит основной домен: `leontev.pro`.

1. В GitHub откройте **Settings → Pages → Custom domain**, укажите `leontev.pro` и сохраните.
2. В DNS-зоне REG.RU добавьте четыре записи типа **A** для `@`:
   - `185.199.108.153`
   - `185.199.109.153`
   - `185.199.110.153`
   - `185.199.111.153`
3. Добавьте запись **CNAME**: `www` → `abkx12-hub.github.io`.
4. Удалите парковочные и конфликтующие записи `A`, `AAAA` или `CNAME` для `@` и `www`.
5. Когда GitHub подтвердит домен, включите **Enforce HTTPS** в Settings → Pages.

DNS может обновляться от нескольких минут до 24 часов. После этого основной адрес сайта — <https://leontev.pro>, а `www.leontev.pro` будет вести на него.

## Как проверить сайт после публикации

Откройте адрес сайта на компьютере и телефоне. Проверьте, что открываются кнопки звонка, почты и WhatsApp, а фотография или fallback с инициалами отображаются корректно.

Если после слияния PR сайт выглядит как старая версия, откройте адрес с параметром проверки, например `https://leontev.pro/?v=latest`, или обновите страницу с очисткой кэша. GitHub Pages иногда отдаёт старую HTML-версию несколько минут после merge.

Контакты редактируются непосредственно в `index.html`.
